
# Authentication System with Role-Based Dashboards

## 1. Database (Supabase migration)

Create three tables with RLS:

- `app_role` enum: `individual`, `researcher`, `enterprise`
- `profiles`: `id uuid PK FK auth.users`, `role app_role`, `full_name text`, `avatar_url text`, `created_at`
- `researcher_profiles`: `id uuid PK`, `user_id uuid FK profiles UNIQUE`, `institution_name`, `publication_url`, `research_field`
- `enterprise_profiles`: `id uuid PK`, `user_id uuid FK profiles UNIQUE`, `company_name`, `company_size`, `website`, `industry`

RLS: users can SELECT/INSERT/UPDATE their own rows (`auth.uid() = id` / `user_id`).
GRANTs for `authenticated` + `service_role`.

Enable Google OAuth via `supabase--configure_social_auth` with `providers: ["google"]`.

## 2. Auth wiring

- Configure `attachSupabaseAuth` in `src/start.ts` (functionMiddleware).
- Add `src/routes/_authenticated/route.tsx` (ssr:false gate, redirect to `/signin`).
- Add root `onAuthStateChange` listener invalidating router/query cache.

## 3. Pages

- `/signup` — 3-step wizard (role → credentials → role-specific fields). Email/password signup + Google button. After signup, insert into `profiles` + role table, redirect to `/dashboard`.
- `/signin` — email/password + Google, forgot password link.
- `/reset-password` — public route to set new password from recovery link.
- `/_authenticated/dashboard.tsx` — fetch profile via serverFn, render Individual/Researcher/Enterprise dashboard variant by role.

## Technical notes

- Use `requireSupabaseAuth` for any serverFn fetching profile data.
- Dashboard page uses client query (`useQuery` + `useServerFn`) — the `_authenticated` layout is `ssr:false` so it's safe.
- Google OAuth uses `lovable.auth.signInWithOAuth("google", ...)` from `@/integrations/lovable` (auto-generated by `configure_social_auth`).
- Keep existing styling tokens (`bg-neuro-gradient`, `glow`, etc.).
